# ✅ Web-48 解題說明 — 團結一致

## 題目描述

這是一個 UNION-Based SQL Injection 的基礎關卡。本題的重點在於「尋找查詢欄位數量」與「尋找回顯點」。題目已經提供了機密情報：系統內有一張表叫做 `level1_flag`，裡面有一個欄位 `flag_text`。駭客需要透過測試確認網頁查詢的欄位數量，並利用 UNION 操作符將機密 Flag 印在畫面上。

## 建議工具

* 瀏覽器網址列或開發者工具（F12 → Console）

## 解題步驟

### 步驟一：尋找破綻 (Find the Injection Point)

**動作：** 在輸入框中輸入單引號 `'`

**背後原理：** * 正常的 SQL 語句：`WHERE sid = '輸入值'`
* 輸入單引號後變成：`WHERE sid = '''`
* 引號不配對會導致 SQL 語法錯誤

**觀察重點：** 如果畫面出現紅色的「Database Error」訊息，證明網頁沒有過濾輸入，你的單引號成功被當作 SQL 程式碼執行。

### 步驟二：確認欄位數量 (Determine Column Count)

**鐵規則：** UNION 前後兩個查詢的「欄位數量」必須一模一樣。

**動作：** 依序輸入以下 Payload：

```sql
' ORDER BY 1 --
' ORDER BY 2 --
' ORDER BY 3 --
' ORDER BY 4 --
```

**背後原理：** * `ORDER BY 數字` = 用第幾個欄位排序
* `--` = SQL 註解符號，廢掉後面多餘的引號

**觀察重點：**
* 測試 `1`、`2`、`3` 時，畫面皆顯示「查無此人」（語法正確，正常執行）。
* 測試到 `4` 時，系統報錯（因為沒有第 4 欄）。

**結論：** 原始查詢有 **3 個欄位**。

### 步驟三：尋找回顯點 (Test for Reflection)

**動作：** 輸入一個不存在的學號並接上包含 3 個欄位的 UNION 查詢：

```sql
nonexistent' UNION SELECT '測試1', '測試2', '測試3' --
```

**背後原理：** * 前半段「nonexistent」絕對找不到任何結果。
* 最終結果只會剩下後半段 UNION 出來的假資料。

**觀察重點：** 畫面成功印出了「測試1」、「測試2」、「測試3」。這證明我們成功控制了網頁的輸出！這三個位置就是等一下要用來「塞入 Flag」的破口。

### 步驟四：奪旗 (Capture The Flag)

根據題目提供的情報，我們已經知道機密表名為 `level1_flag`，欄位為 `flag_text`。

**最終奪旗動作：** 將剛剛的「測試」字眼，替換成我們要查詢的目標：

```sql
nonexistent' UNION SELECT '第一欄', flag_text, '第三欄' FROM level1_flag --
```

**觀察重點：** 畫面成功印出機密資訊，格式包含 `flag{L3v3l_1_un10n_b4s1cs}`。

## 其他知識

### UNION SQL Injection 的進階應用

* **通用 Payload：** `UNION SELECT NULL, NULL, NULL...` 可以快速測試欄位數量，因為 `NULL` 能適應任何資料型態，可以避免因型態不匹配而報錯。
* **轉換資料型態：** 若欄位要求特定型態，可以使用 `CAST()` 或 `CONVERT()` 來轉換。

### 防禦方法

* **使用 Prepared Statements：** 將 SQL 語法與資料分離，是最有效的防禦方式（如 PDO 的 `prepare()` 與 `execute()`）。
* **輸入驗證與過濾：** 驗證輸入是否符合預期格式（例如學號應該是 `s26XXX` 格式）。
* **錯誤處理：** 避免在使用者界面上顯示詳細的資料庫錯誤訊息，改為顯示友善的提示。

## 驗證與常見卡點

* **驗證方式：** 最終輸出應符合 `flag{...}` 格式。
* **常見卡點：** 忘記加註解符號 `--`，導致語句不完整。
* **常見卡點：** UNION 前後欄位數量不一致，本題必須湊齊 3 個欄位。
* **常見卡點：** 沒有讓前面的查詢變成空（忘記用 `nonexistent`），導致原本的資料干擾了視線。

## 學習重點

* **ORDER BY 測試的實用性：** 這是探測欄位數量的標準流程，適用於所有 SQL Injection。
* **UNION 拼接的核心精神：** 駭客不僅能查詢，還能決定輸出的欄位數量與位置，將內部資料完美融合到正常網頁中。
